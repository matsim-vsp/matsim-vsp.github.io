## Project description

Project C - Integration of data and models towards the AgiMo Digital Twin

Transport models typically need a lot of time and effort (several person months) to be set up. This
is true even if a model already exists and only needs to be updated to more recent data. Also,
models are normally built for a “typical day”, meaning that they do not follow different demand levels
over the seasons, during school vacations, etc. This means that giving model-based answers to
transport-related policy questions typically takes at least a couple of months.
In this project, we will address these issues by moving towards the open data AgiMo Digital
Twin of the real world transportation system. The aspiration is a digital twin for planning purposes,
implying that the time scale of the system reaction is several months, and the feedback loop into
the real system is not a technical channel, but by feeding the model results into the political process, 
in this CRC via C4 to C2. The starting point will be a simulation of a region, in combination
with a setup that updates (‘assimilates’) that simulation to additional data regularly, initially once
per week. The results will be uploaded to the data infrastructure of INF, leading to automatically
downloadable assimilated data. The ultimate goal will be to have such a system track real world
developments, e.g., differences between weekdays, reduced trafﬁc during school holidays, reactions to street closures or special events, etc.
We will leverage a data feed previously utilized in the context of COVID-19-related simulations,
which provides daily out-of-home durations at zipcode level (cf. INF for the data procurement) –
this will be used to increase or decrease activity participation levels accordingly. Another data
feed, with much smaller sample size but much better spatial resolution, will provide changes in
activity participation by activity type, thus allowing to adjust not just the overall participation levels,
but to do this by activity type. Methods from C7 will be used to give the data assimilation process
a theoretical foundation. Work in A6 will be tightly integrated to more realistically model the active
modes walking and cycling.
Projects such as B2, B7, C5, and C6 will use the automatically uploaded data via INF as input
to their own work. Besides the “digital twin” research agenda, this project thus serves as an
umbrella project for many of the data and modeling projects in this CRC (such as B3 or A7)
and as a use case for stress-testing the data and modeling techniques developed in INF.
The main research question of this project is:
- Which data items and methodologies need to be combined, and in which way, to build a credible planning digital twin of the real world transportation system?
